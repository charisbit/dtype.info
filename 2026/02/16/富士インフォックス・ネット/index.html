富士インフォックス・ネット

富士インフォックス・ネット面接準備資料

自己じこ紹介しょうかい（2-3分ふん）

本日ほんじつはお忙しいいそがしい中なか、貴重なきちょうなお時間じかんをいただき、ありがとうございます。盛偉と申しもうします。

私わたしは中国ちゅうごく出身しゅっしんで、2016年ねんに来らい日にちしてから8年ねん以上いじょう、日本にほんでシステムしすてむエンジニアえんじにあとして働いてはたらいております。武漢ぶかん大学だいがくで電気でんき工学こうがくを、浙江せっこう大学だいがく大学院だいがくいんでシステムしすてむ分析ぶんせきと統合とうごうを学びまなび、来らい日にち後ごはMonotaRO、エクレシア企画きかく、そして現在げんざいのSynXと、様々なさまざまな企業きぎょうでフルふるスタック開発かいはつに携わってたずさわってまいりました。

技術ぎじゅつ面めんでは、Python、Java、JavaScript、Golangなど多様なたような言語げんごを扱いあつかい、バックエンドからフロントエンド、データでーた基盤きばん、そして最近さいきんでは機械きかい学習がくしゅうやLLMを活用かつようしたシステムしすてむ開発かいはつまで幅広くはばひろく経験けいけんしております。特にとくに、MonotaROではGCPを活用かつようしたデータでーたプラットフォームの構築こうちく、現在げんざいのSynXではAGV制御せいぎょシステムしすてむやLLM統合とうごう監視かんしシステムしすてむの開発かいはつをプロジェクトぷろじぇくとリーダーりーだーとして主導しゅどうしており、技術ぎじゅつ力りょくだけでなくマネジメントまねじめんと経験けいけんも積んでつんでおります。

御おん社しゃに興味きょうみを持ったもった理由りゆうは、32年間ねんかんの実績じっせきに裏付けうらづけられた技術ぎじゅつ力りょくと、Spring BootやJavaを活用かつようした自社じしゃサービスさーびす開発かいはつ、そして月つき平均へいきん残業ざんぎょう9時間じかんという働きはたらきやすい環境かんきょうに魅力みりょくを感じたかんじたからです。私わたしのこれまでの経験けいけんを活かしいかしながら、要件ようけん定義ていぎから保守ほしゅまで一貫いっかんしたプロジェクトぷろじぇくとに携わりたずさわり、御おん社しゃの更ふけなる発展はってんに貢献こうけんしたいと考えてかんがえております。

本日ほんじつはどうぞよろしくお願いねがいいたします。

予想よそう質問しつもんと回答かいとう

Q1: なぜ富士ふじインいんフォックス・ネットねっとに興味きょうみを持ったもったのですか？

御おん社しゃに興味きょうみを持ったもった理由りゆうは三さんつございます。

まず、32年間ねんかんという長いながい実績じっせきと富士通ふじつうコアこあパートナーぱーとなーとしての信頼しんらい性せいです。私わたしはこれまで様々なさまざまな規模きぼの企業きぎょうで働いてはたらいてきましたが、長期ちょうき的にてきに安定あんていした技術ぎじゅつ基盤きばんを持つもつ企業きぎょうで、自分じぶんのスキルすきるをより深めふかめたいと考えてかんがえております。

次につぎに、自社じしゃサービスさーびす開発かいはつへの取り組みとりくみです。「スクールすくーるアシストあしすと昴すばる」や「BI21」といった自社じしゃプロダクトの開発かいはつに携われるたずさわれることで、単なるたんなる受託じゅたく開発かいはつではなく、エンドえんどユーザーゆーざーのニーズにーずを直接ちょくせつ理解りかいし、価値かちのあるサービスさーびすを作りつくり上げるあげる経験けいけんを積めるつめると感じかんじました。

最後にさいごに、働きはたらきやすい環境かんきょうづくりです。月つき平均へいきん残業ざんぎょう9時間じかん、在宅ざいたく勤務きんむ制度せいど、港みなと区くワークライフらいふバランスばらんす推進すいしん企業きぎょう認定にんていなど、技術ぎじゅつ者しゃが長期ちょうき的にてきに成長せいちょうできる環境かんきょうが整ってととのっていると感じかんじました。

Q2: これまでの経験けいけんで最ももっとも印象いんしょうに深いふかいプロジェクトぷろじぇくとは？

最ももっとも印象深いいんしょうぶかいのは、現在げんざいのSynXで開発かいはつしたLLM活用かつよう型かた監視かんし自動じどう化かシステムしすてむ「Walkure Operator」です。

このプロジェクトぷろじぇくとでは、Zabbixと複数ふくすうのLLM（Mistral、Mixtral等とう）を統合とうごうし、アラートの自動じどう要約ようやくや原因げんいん推定すいてい、予知よち保全ほぜんを行うおこなうシステムしすてむを一いちから設計せっけい・実装じっそうしました。プロジェクトぷろじぇくとリーダーりーだーとして、技術ぎじゅつ選定せんていからアーキテクチャ設計せっけい、開発かいはつチームちーむの管理かんりまで担当たんとういたしました。

特にとくに難しかったむずかしかったのは、異なることなるLLMモデルもでるの特性とくせいを理解りかいし、MCP（Model Context Protocol）を用いてもちいてモジュールもじゅーる化かすることで、将来しょうらい的なてきな拡張かくちょう性せいを確保かくほすることでした。結果けっかとして、段階だんかい的にてきにLLMを導入どうにゅうできる仕組みしくみを構築こうちくし、運用うんよう監視かんしの自動じどう化かと品質ひんしつ向上こうじょうを実現じつげんできました。

このプロジェクトぷろじぇくとを通じてつうじて、最新さいしん技術ぎじゅつを実務じつむに活用かつようする能力のうりょくと、チームちーむをまとめてプロジェクトぷろじぇくとを成功せいこうに導くみちびく力ちからを身みにつけました。

Q3: Java/Spring Bootの経験けいけんはありますか？

はい、Javaの経験けいけんは大学だいがく時代じだいから含めるふくめると10年ねん以上いじょうございます。

実務じつむでは、MonotaRO時代じだいにJavaでのWebアプリケーションあぷりけーしょん開発かいはつ、そして最近さいきんのSynXでの統合とうごうプロジェクトぷろじぇくと管理かんりツールつーる「FAQT」Ver.1.5開発かいはつにおいて、Java11とSpring Bootを活用かつようしたバックエンド開発かいはつに携わりたずさわりました。

Spring Bootでは、RESTful APIの設計せっけい・実装じっそう、JPA/Hibernateを用いたもちいたデータベースでーたべーす連携れんけい、Spring Securityによる認証にんしょう・認可にんか機能きのうの実装じっそうなどを経験けいけんしております。また、Maven/Gradleを用いたもちいたビルド管理かんり、JUnitでのテストてすと駆動くどう開発かいはつも実践じっせんしてまいりました。

御おん社しゃの自社じしゃサービスさーびす開発かいはつでも、これまでの経験けいけんを活かしいかしながら、より高こう品質ひんしつで保守ほしゅ性せいの高いたかいシステムしすてむ開発かいはつに貢献こうけんできると考えてかんがえております。

Q4: チームちーむでの開発かいはつ経験けいけんについて教えておしえてください。

私わたしはこれまで様々なさまざまな規模きぼのチームちーむで開発かいはつ経験けいけんを積んでつんでまいりました。

MonotaRO時代じだいは5-8名めいのデータでーたプラットフォームチームちーむで、主におもにシニアしにあエンジニアえんじにあとして技術ぎじゅつ的なてきなリードりーどを担当たんとうしました。GCPを活用かつようしたデータでーたパイプぱいぷラインらいん構築こうちくでは、チームちーむメンバーめんばーのスキルすきるレベルれべるに合わせてあわせて作業さぎょうを分担ぶんたんし、定期ていき的なてきなコードこーどレビューれびゅーと技術ぎじゅつ共有きょうゆうを通じてつうじて品質ひんしつを担保たんぽしました。

現在げんざいのSynXでは、プロジェクトぷろじぇくとリーダーりーだーとして3-5名めいのチームちーむを管理かんりしております。特にとくにAGV制御せいぎょシステムしすてむ開発かいはつでは、要件ようけん定義ていぎから設計せっけい、実装じっそう、テストてすとまでの全ぜん工程こうていでチームちーむをマネジメントまねじめんとし、毎週まいしゅうの進捗しんちょく会議かいぎとスクラムすくらむ手法しゅほうを取り入れたとりいれたアジャイル開発かいはつを実践じっせんしています。

コミュニケーションこみゅにけーしょんでは、技術ぎじゅつ的なてきな議論ぎろんだけでなく、メンバーめんばーのモチベーション管理かんりや成長せいちょう支援しえんも重視じゅうししており、チームちーむ全体ぜんたいのパフォーマンスぱふぉーまんす向上こうじょうに努めてつとめております。

Q5: 将来しょうらいのキャリアきゃりあプランぷらんについて教えておしえてください。

私わたしの将来しょうらいのキャリアきゃりあプランぷらんは、技術ぎじゅつ的なてきな専門せんもん性せいとマネジメントまねじめんと能力のうりょくの両方りょうほうを高めるたかめることです。

短期たんき的にてきには、御おん社しゃでJava/Spring Bootを中心ちゅうしんとした自社じしゃサービスさーびす開発かいはつに携わりたずさわり、要件ようけん定義ていぎから保守ほしゅまで一貫いっかんしたプロジェクトぷろじぇくと経験けいけんを積みつみたいと考えてかんがえております。特にとくに、「スクールすくーるアシストあしすと昴すばる」のような教育きょういく分野ぶんやのシステムしすてむは社会しゃかい的てき意義いぎも大きくおおきく、やりがいを感じかんじながら技術ぎじゅつ力りょくを向上こうじょうさせたいです。

中なか長期ちょうき的にてきには、プロジェクトぷろじぇくとマネージャーまねーじゃーやテックリードとして、より大だい規模きぼなプロジェクトぷろじぇくとの責任せきにん者しゃを務めつとめたいと考えてかんがえております。また、AI・機械きかい学習がくしゅうやクラウド技術ぎじゅつといった先端せんたん分野ぶんやの知識ちしきを深めふかめ、御おん社しゃの新規しんき事業じぎょう開発かいはつにも貢献こうけんしたいです。

最終さいしゅう的にてきには、技術ぎじゅつ者しゃとしての専門せんもん性せいを活かしいかしながら、チームちーむや組織そしき全体ぜんたいの技術ぎじゅつ力りょく向上こうじょうに貢献こうけんできるリーダーりーだーになりたいと考えてかんがえております。

私わたしからの質問しつもん

技術ぎじゅつ・開発かいはつ関連かんれん

  1. 開発かいはつチームちーむの構成こうせいと開発かいはつプロセスぷろせすについて
    現在げんざいの開発かいはつチームちーむはどのような構成こうせいになっておりますでしょうか？また、アジャイル開発かいはつやDevOpsなどの開発かいはつ手法しゅほうは取り入れとりいれられていますか？

  2. 技術ぎじゅつスタックと技術ぎじゅつ選定せんていについて
    現在げんざい主におもに使用しようされている技術ぎじゅつスタックを教えておしえてください。また、新しいあたらしい技術ぎじゅつの導入どうにゅう検討けんとうはどのような基準きじゅんで行わおこなわれていますか？

  3. 自社じしゃサービスさーびす開発かいはつの比重ひじゅうについて
    現在げんざい、受託じゅたく開発かいはつと自社じしゃサービスさーびす開発かいはつの比重ひじゅうはどの程度ていどでしょうか？今後こんご、自社じしゃサービスさーびす開発かいはつにより注力していく予定よていはありますか？

  4. 技術ぎじゅつ的なてきな成長せいちょう支援しえんについて
    エンジニアえんじにあのスキルすきるアップあっぷやキャリアきゃりあ開発かいはつのためにどのような支援しえん制度せいどがございますか？資格しかく取得しゅとく支援しえんや社しゃ外がい研修けんしゅうの機会きかいはありますでしょうか？

業務ぎょうむ・環境かんきょう関連かんれん

  1. リモートワークの実際じっさいの運用うんようについて
    在宅ざいたく勤務きんむ制度せいどは実際じっさいにどの程度ていど活用かつようされていますか？チームちーむコミュニケーションこみゅにけーしょんはどのように行わおこなわれていますか？

  2. プロジェクトぷろじぇくとアサインメントについて
    新しくあたらしく入社にゅうしゃしたエンジニアえんじにあは、どのようなプロセスぷろせすでプロジェクトぷろじぇくとにアサインあさいんされますか？希望きぼうや適性てきせいは考慮こうりょされますでしょうか？

  3. 評価ひょうか制度せいどとキャリアパスについて
    エンジニアえんじにあの評価ひょうかはどのような基準きじゅんで行わおこなわれますか？また、技術ぎじゅつ者しゃとしてのキャリアきゃりあアップあっぷの道筋みちすじはどのようになっていますか？

  4. 会社かいしゃの今後こんごのビジョンびじょんについて
    地方ちほう創生やDX推進すいしんといった新しいあたらしい分野ぶんやへの展開てんかいについて、具体ぐたい的にてきにはどのような取り組みとりくみを計画けいかくされていますか？
